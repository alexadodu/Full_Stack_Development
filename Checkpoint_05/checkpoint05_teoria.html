1. Qu es una condicional?

1.1. Definicin

Una condicional en Python es una estructura de control de flujo que permite tomar decisiones basadas en condiciones especficas. Estas condiciones se evalan como verdaderas o falsas y, dependiendo del resultado, se ejecuta un bloque de cdigo u otro.

1.2. Tipos de condicionales

Podramos catalogar las condicionales en tres tipos: simples, compuestas y anidadas. Vamos a ir viendo ahora cada una de ellas para entenderlas.

1.2.1. Estructura condicional simple

Las condicionales simples se emplean para ejecutar un conjunto de instrucciones solamente si se cumple una condicin determinada. Podemos observar en el esquema que el rombo representa la condicin y tenemos dos opciones que podemos tomar. Si la condicin devuelve un resultado verdadero, ejecutar las opciones que se encuentren en su interior. Si, por el contrario, es falso, no realizar dicha accin.

Sintaxis

Para la sintaxis de una condicional simple se emplea la sentencia if. El condicional if se utiliza para ejecutar un bloque de cdigo si una condicin es verdadera. Despus de la sentencia if, se aaden dos puntos : y, en la siguiente lnea, hay que aadir indentacin.

if <condicin>:
    <bloque_de_cdigo_con_indentacin>

Ejemplo

En este primer ejemplo, podemos observar que, al ser la edad de 23 menor que 26, se imprime por pantalla que puede acceder a solicitar el Carnet Joven. Observar la sintaxis con los dos puntos y la indentacin.

In[17]:

if edad < 26:
    print("Puedes acceder a solicitar el Carnet Joven.")

Puedes acceder a solicitar el Carnet Joven.

Sin embargo, en este otro ejemplo, en el cual no se cumple la edad, puesto que 26 no es menor que 26, sino igual, no se imprime nada.

In[20]:
edad = 26
if edad < 26:
    print("Puedes acceder a solicitar el Carnet Joven.")

Por tanto, slo se realiza una accin cuando la condicin es verdadera. Cuando es falsa, no se toma ninguna accin. Pero, cmo podemos tomar decisiones cuando estamos en este ltimo caso? Pasemos al siguiente tipo de condicional.

1.2.2. Estructura condicional compuesta

Las condicionales compuestas se emplean para ejecutar un conjunto de instrucciones cuando hay mltiples condiciones que deben evaluarse. Si ahora nos fijamos en el esquema, vemos que podemos tomar decisiones cuando la condicin es falsa.

Sintaxis

Se emplea la sentencia if junto con la sentencia else. El condicional else se utiliza para ejecutar un bloque de cdigo si ninguna de las condiciones anteriores se cumple. Aqu tambin se aaden dos puntos : despus de la condicional y la indentacin en las siguientes lneas, con las instrucciones a realizar.

if <condicin>:
    <bloque_de_cdigo_con_indentacin>
else:
    <bloque_de_cdigo_con_indentacin>

Ejemplo

Siguiendo con el ejemplo anterior, en este caso se imprime por pantalla un mensaje cuando la edad no es menor de 26 aos:

    print("Puedes acceder a solicitar el Carnet Joven.")
else:
    print("Lo siento, no puedes acceder a solicitar el Carnet Joven.")

Lo siento, no puedes acceder a solicitar el Carnet Joven.

1.2.3. Estructuras condicionales anidadas

Las condicionales anidadas se emplean cuando nos interese que una condicional est dentro de otra, es decir, tomar decisiones dentro de otras decisiones. Se pueden incluir condicionales simples o compuestas dentro de ellas, como se puede observar en el siguiente esquema. La principal condicin que podemos ver es una estructura compuesta y la segunda condicin es en este caso, una estructura simple.

Es comn que se representen este tipo de condiciones anidadas aunque esto es un simple ejemplo en comparacin con lo que se puede ver normalmente en cualquier cdigo.

Sintaxis

En cuanto a su sintaxis, adems de las condicionales if y else, se puede emplear el condicional elif. El condicional elif se utiliza para evaluar mltiples condiciones despus de la primera condicin if. Si la primera condicin no se cumple, se evala la siguiente condicin con elif.

La condicional elifse puede emplear tantas veces se requiera, sin embargo, la condicional else es opcional.

Recordar los dos puntos despus de la condicin y la indentacin en el cdigo!

    <bloque_de_cdigo_con_indentacin>

    <bloque_de_cdigo_con_indentacin>
else:
    <bloque_de_cdigo_con_indentacin>

El for es un tipo de bucle parecido al while, pero con ciertas diferencias. La principal es que el nmero de iteraciones de un for esta definido de antemano, mientras que en un while no. 3. Qu es una lista por comprensin en Python?

3.1. Definicin

Una comprensin de lista, tambin conocida como list comprehension en ingls, es una forma concisa y elegante de crear listas en Python, en una sola lnea de cdigo. Permite crear listas utilizando una sintaxis ms compacta que los bucles for tradicionales y utilizando, opcionalmente, usando condicionales if.

3.2. Uso

Pero, cundo usar las listas por comprensin?

  1. Transformacin de datos: Cuando necesites transformar los elementos de una lista de una forma especfica, como convertir todos los elementos a maysculas, multiplicarlos por un valor constante, etc.

  2. Filtrado de datos: Cuando desees filtrar los elementos de una lista segn ciertos criterios, como eliminar elementos que no cumplan con cierta condicin o seleccionar solo los elementos que satisfacen ciertos requisitos.

  3. Construccin de listas de forma eficiente: Las listas por comprensin suelen ser ms eficientes y concisas que los bucles for tradicionales, por lo que son una buena opcin cuando deseas construir listas de manera rpida y clara.

  4. Legibilidad del cdigo: En muchos casos, una lista por comprensin puede hacer que el cdigo sea ms claro y fcil de entender, especialmente cuando se utiliza para operaciones simples y directas en los elementos de una lista.

3.3. Sintaxis

La sintaxis sera la siguiente:

nueva_lista = [<nombre_expresion> for <variable> in <iterable> [if <condicin>]]

Qu significa cada uno de los componentes?

  • nueva_lista: Es el nombre de la nueva lista.
  • nombre_expresion: Es una expresin que define el valor de cada elemento en la lista resultante.
  • El segundo componente for <variable> in <iterable> sera el correspondiente a un bucle for, que acabamos de ver en el apartado anterior.
  • El tercer componente [if <condicin>], que es opcional, sera el condicional if, que hemos visto en el apartado 1.

Importante entonces aqu: la lista por comprensin sigue siendo una lista, por tanto, es un tipo de objeto. Por ello, lleva corchetes [ ] y se almacena en un objeto.

4. Qu es un argumento en Python?

4.1. Definicin

Los argumentos en Python son los valores que se pasan a una funcin cuando se la llama. Estos valores se utilizan dentro de la funcin para realizar operaciones o clculos.

Los argumentos permiten que las funciones sean ms flexibles y reutilizables al permitir que se ejecuten con diferentes conjuntos de datos o configuraciones.

El manejo de argumentos en funciones Python es una tcnica fundamental que cualquier desarrollador debe conocer. En Python, la flexibilidad y adaptabilidad que ofrecen las funciones en el manejo de argumentos lo convierten en un lenguaje de programacin poderoso para el desarrollo de aplicaciones complejas.

4.2. Tipos de argumentos

Los argumentos pueden ser de diferentes tipos y se pueden pasar de diferentes maneras a las funciones. Vamos a ver los tipos principales: argumentos posicionales, argumentos de palabra clave, argumentos por defecto y argumentos variables.

4.2.1. Argumentos posicionales

Cuando enviamos argumentos a una funcin, estos se reciben por orden en los parmetros definidos. Se dice, por tanto, que son argumentos por posicin.

El uso de argumentos posicionales en Python brinda flexibilidad a las funciones al permitir una llamada ms simple y ordenada para una variedad de argumentos. Adems, se pueden definir valores por defecto para los argumentos en caso de no ser enviados en la llamada a la funcin.

Un ejemplo sera:

In[24]:
def resta(a, b):
    return a - b

resta(10, 5)
Out[24]:
5

4.2.2. Argumentos por palabra clave

Sin embargo, es posible evadir el orden de los parmetros si indicamos durante la llamada que valor tiene cada parmetro a partir de su nombre. Cuando programamos en Python, no solo debemos preocuparnos por la lgica de nuestro cdigo, sino tambin por su flexibilidad y adaptabilidad. Una de las formas en las que podemos lograr esto es mediante el uso de argumentos con palabras clave.

Los argumentos con palabras clave son una herramienta muy til en Python para hacer nuestro cdigo ms legible y adaptable. Para utilizarlos en nuestras funciones, basta con definir los argumentos de la forma argumento=valor. Con esta tcnica, podemos hacer que nuestras funciones se adapten a diferentes situaciones sin tener que cambiar su lgica interna.

Por tanto, los argumentos con palabras clave son aquellos que se definen mediante una palabra clave seguida de un signo igual y su valor correspondiente. En el siguiente ejemplo, fijarse en que el orden de los argumentos est cambiado, ahora el argumento b va primero.

Mala praxis con argumentos por defecto

CUIDADO! Es importante tener en cuenta que los valores por defecto solo se evalan una vez, al definir la funcin. Esto significa que si se usa un objeto mutable (por ejemplo, una lista o un diccionario) como valor por defecto, se pueden producir resultados inesperados.

En el siguiente ejemplo, se usar una lista como valor predeterminado para el parmetro lista:

    return lista

El uso de listas (o cualquier otro objeto mutable) como valor predeterminado en los parmetros de una funcin en Python se considera una mala prctica por varias razones:

  1. Comparticin de estado entre llamadas: Cuando una lista se utiliza como valor predeterminado, se crea una nica instancia de la lista en tiempo de definicin de la funcin. Esto significa que todas las llamadas posteriores a la funcin compartirn la misma instancia de la lista. Esto puede llevar a resultados inesperados si la lista se modifica en una llamada y los cambios afectan a llamadas posteriores.

  2. Efectos secundarios inesperados: Debido a la comparticin de estado, las modificaciones realizadas en la lista dentro de una llamada pueden persistir entre llamadas subsiguientes, lo que puede conducir a efectos secundarios inesperados y difciles de depurar.

  3. Dificultad para rastrear el estado: Al utilizar valores predeterminados mutables, puede resultar difcil rastrear el estado de la funcin y entender cmo se estn modificando los datos a lo largo del tiempo, especialmente en cdigo ms grande y complejo.

Por tanto, como ya he mencionado, para evitar estos problemas, es mejor utilizar None como valor predeterminado para los argumentos mutables y luego inicializar la lista (o cualquier otro objeto mutable) dentro de la funcin si es necesario. De esta manera, se garantiza que cada llamada a la funcin tenga su propia instancia de la lista y se evitan los efectos secundarios no deseados.

5.2. Ventajas y desventajas de las funciones lambda

Las funciones lambda son importantes en Python por varias razones:

  1. Concisin de cdigo: Las funciones lambda permiten definir funciones de forma ms concisa y compacta que las funciones regulares, especialmente cuando se trata de funciones simples que realizan operaciones bsicas.

  2. Legibilidad del cdigo: A menudo, una funcin lambda puede hacer que el cdigo sea ms legible al colocar la lgica de la funcin directamente en el lugar donde se utiliza, sin la necesidad de definir una funcin separada con un nombre descriptivo.

  3. Flexibilidad: Las funciones lambda pueden utilizarse en lugares donde se requieren funciones como argumentos, como en las funciones map(), filter() y sorted(), lo que permite una mayor flexibilidad en el diseo del cdigo.

  4. Programacin funcional: Las funciones lambda son una parte importante de la programacin funcional en Python, que se centra en el uso de funciones como ciudadanos de primera clase y en la aplicacin de operaciones funcionales como map, filter y reduce.

Sin embargo, tambin tiene ciertas limitaciones a tener en cuenta:

  1. Simplicidad: las funciones Lambda solo pueden tener una expresin, por lo que no pueden manejar tareas ms grandes.
  2. Legibilidad: si se usan en exceso o se hacen demasiado complejos, pueden hacer que el cdigo sea ms difcil de entender.

5.3. Sintaxis

La sintaxis bsica de una funcin lambda es la siguiente:

  • A diferencia de las funciones habituales que definimos usando la palabra clave def, las funciones lambda las definimos usando la palabra clave lambda.
  • Seguido de uno o ms argumentos (al igual que los argumentos de funciones normales).

  • La expresin es lo que devolver la funcin cuando se llame.

Una funcin lambda no tiene un nombre y, por lo tanto, salvo que sea asignada a una variable, es totalmente intil:

Una funcin lambda puede tomar cualquier nmero de argumentos, pero solo puede tener una expresin.
